# Engineering · 1주차 사전 과제 — 모래시계 (Computational Thinking)

> 대상 사물: **모래시계**
> CT 4단계: 분해 → 패턴 인식 → 추상화 → 플로우차트, 그리고 작동 프로토타입
> 분해·패턴·추상화는 *모래시계 그 자체*를 다루고, 플로우차트·프로토타입은 그 추상화에서 파생된 **새 인터페이스(모래시계 타이머)** 를 다룹니다.

## 1. 분해안 (Decomposition)

복잡한 시스템을 가장 작은 요소로 쪼갠 뒤, 그것들이 어떻게 결합해 상위 목적을 달성하는지 하향식(Top-down) 트리로 파악한다.
재질·색상 같은 겉모습은 배제하고, **입력 → 저장 → 처리 → 출력 → 종료** 라는 *사용 시퀀스*가 트리에 온전히 드러나도록 했다.
가장 저수준은 모두 **수치 또는 참/거짓**으로 측정 가능하다.

### 1-1. 텍스트 트리

```
[고수준 · 목적 한 문장]
모래시계 : 중력을 동력 삼아, 고정된 양의 모래가 일정한 속도로 떨어지는 데 걸리는 시간을 시각적으로 측정하는 장치

├─ [중수준 1] 입력 — 물리적 반전(뒤집기)으로 측정을 초기화·시작한다
│   ├─ 저수준 1-A · 반전 상태   isFlipped            : True / False        (상태)
│   ├─ 저수준 1-B · 반전 이벤트 flipEvent            : 발생(1) / 없음(0)    (입력 이벤트=트리거)
│   └─ 저수준 1-C · 반전 시각   flipTimestamp        : 측정 시작 기준점(수치)(입력 이벤트)
│
├─ [중수준 2] 저장 — 떨어질 모래(에너지)를 상부 챔버에 적재한다
│   ├─ 저수준 2-A · 전체 모래량 totalSand (상수)     : 100 % (= 측정 가능한 시간의 최댓값, 예 180초)
│   ├─ 저수준 2-B · 상부 모래량 topSand  (독립 변수) : 0 ~ 100 %   ← 시스템의 유일한 독립 상태변수
│   └─ 저수준 2-C · 하부 모래량 bottomSand(파생)     : = 100 − topSand  (top + bottom = 100 %)
│
├─ [중수준 3] 처리 — 중력이 좁은 목(오리피스)으로 모래를 일정 속도로 내려보낸다
│   ├─ 저수준 3-A · 낙하 속도 flowRate (상수)        : 단위시간당 %  (목 지름·입자 크기로 고정)
│   ├─ 저수준 3-B · 단위시간 흐름 tick (반복 단위)    : dt초마다 topSand 를 1회 갱신  ← 반복 시퀀스(loop)
│   ├─ 저수준 3-C · 경과 시간 elapsed (파생)          : (100 − topSand) ÷ flowRate (수치, 초)
│   └─ 저수준 3-D · 잔여 시간 remaining (파생)        : topSand ÷ flowRate (수치, 초)
│
├─ [중수준 4] 출력 — 모래 높이·흐름의 변화로 시간을 시각적으로 표시한다
│   ├─ 저수준 4-A · 상부 기둥 높이 topHeight         : 0 ~ H (수치, mm/px)
│   ├─ 저수준 4-B · 하부 더미 높이 bottomHeight      : 0 ~ H (수치)
│   ├─ 저수준 4-C · 진행률      progress             : elapsed ÷ 총시간 = 0.0 ~ 1.0 (파생)
│   └─ 저수준 4-D · 완료 신호   isDone               : (topSand == 0) → True / False  ← 사용자가 관측하는 출력
│
└─ [중수준 5] 종료 — 상부 모래가 0이 되면 측정이 끝나고 정지한다
    └─ 저수준 5-A · 정지 상태  isIdle                : 흐름 정지(True) / 흐름 중(False)
```

### 1-2. 조작(액션) → 데이터 변화 규칙

> 목적(시간 측정)을 달성하려고 거치는 조작은 **2종**뿐이다. 각 조작이 *어떤 데이터를 어떻게 바꾸는지* 좌변(변하는 값) = 우변(변화 규칙)으로 명시한다.

| 조작 (액션) | 변하는 데이터 | 변화 규칙 | 트리거 시점 |
|---|---|---|---|
| **A. 뒤집기 (flip)** | `topSand`, `bottomSand`, `isFlipped`, `flowRate` | `topSand ← 100`, `bottomSand ← 0`, `isFlipped` 토글, `flowRate` 방향 반전 | 사람이 기기를 뒤집을 때 (외부 입력) |
| **B. 시간경과 (tick, dt초)** | `topSand`, `bottomSand` | `topSand ← max(0, topSand − flowRate·dt)`, `bottomSand ← 100 − topSand` | 작동 중 매 단위시간 (내부 자동) |

**파생값 재계산 인과:** `tick → topSand 갱신 → (elapsed, remaining, progress, isDone) 재계산`
→ remaining/progress/elapsed는 독립적으로 변하지 않고, *오직 topSand가 바뀐 직후* 다시 계산되는 읽기 전용 값이다.

### 1-3. 입력 → 내부상태 → 출력 (LO1 점검)

```
[입력]                  [내부 상태/연산]                         [출력 = 사용자가 보는 것]
뒤집기(flip) ───────▶ topSand·bottomSand 변경, flowRate 반전 ─┐
시간경과(tick) ─────▶ topSand 감소 → 파생값 재계산 ───────────┴─▶ 모래 높이(topHeight/bottomHeight),
                                                                  진행률(progress), 완료 신호(isDone)
```

| 데이터 종류 | 요소 |
|---|---|
| **입력(외부 조작)** | 뒤집기 — `flipEvent`, `flipTimestamp` |
| **상수** | `totalSand`, `flowRate` |
| **독립 상태변수** | `topSand` *(유일)* — 이 값 하나가 시스템 전체 상태를 결정 |
| **파생값(읽기전용)** | `bottomSand`, `elapsed`, `remaining`, `progress`, `isDone` = f(`topSand`) |
| **상태 플래그** | `isFlipped`, `isIdle` (True/False) |
| **출력(사용자가 보는 결과)** | 모래 높이(`topHeight`,`bottomHeight`), `progress`, 완료 신호(`isDone`) — *시각 출력만* |

> **완결성 점검:** 실제 사용 순서 — *뒤집는다(입력) → 모래가 쌓인다(저장) → 흘러내린다(처리) → 높이로 보인다(출력) → 다 떨어지면 멈춘다(종료)* — 가 트리 5개 가지에 1:1로 모두 들어왔다. 빠진 단계 없음.

## 2. 패턴 인식 (Pattern Recognition)

분해된 요소들 사이의 작동 규칙을 **비교 매트릭스**로 정리한다.
1주차 사전 과제이므로 **모래시계 컬럼만** 채우고, 나머지 두 사물은 차주 과제 범위로 비워 둔다.

| 비교 기준 (분해 속성) | **모래시계** | 태엽 오르골 | 기계식 스톱워치 |
|---|---|---|---|
| **동력원 (Energy)** | 중력 위치에너지 — 상부 모래의 무게. 외부 전력 없이, 한 번 적재된 위치에너지를 소모 | *(2주차)* | *(3주차)* |
| **입력 방식 (Trigger)** | 물리적 반전(뒤집기) **1종**. 상↔하 이산 토글 이벤트 | — | — |
| **출력 방식 (Output / Feedback)** | **시각 출력만** — 상/하 모래 높이 변화·흐름. 소리·진동·알림 없음 | — | — |
| **상태 제어 (Control)** | **무조건 연속 흐름**. 시작 후 일시정지 불가, 정지는 모래 소진 시 자동. 리셋·역전은 *반드시 뒤집어야만* 가능 | — | — |
| 측정 대상·단위 | 고정 시간 **1종**(totalSand ÷ flowRate). 눈금 없는 아날로그 | — | — |
| 에너지 저장 형태 | 상부 챔버의 모래 더미(위치에너지). 재충전 = 뒤집기 | — | — |
| 정밀도·제약 | 낙하 속도가 습도·모래량에 따라 변동(오차 큼). 누적·랩(lap) 측정 불가 | — | — |

### 핵심 작동 규칙 (모래시계 한 문장)
> **한 번의 물리적 반전으로 고정량의 에너지를 적재하면, 외부 개입 없이 일정 속도로 에너지가 소모되며, 그 소모 진행도가 곧 경과 시간이 된다.**

### 반복 시퀀스(loop) 식별
- **소모 루프(micro):** 작동 중 매 단위시간 `tick`마다 `topSand ← topSand − flowRate·dt` 를 반복.
- **세션 루프(macro):** 뒤집기 → 소진 → 다시 뒤집기로 측정 전체를 반복.

*(공통 패턴 / 차이 패턴은 세 사물을 나란히 놓는 차주 이후에 도출 — 1주차에서는 작성 생략 가능)*

## 3. 추상화 (Abstraction)

물리적 형태('모래알의 양', '중력', '유리관')를 걷어내고, 디지털 UI에서 다룰 수 있는 보편 속성으로 바꾼다.

### 추상화 모델 — 한 줄 문장
> **외부 입력으로 일정량의 에너지를 충전한 뒤, 상태 전환 신호에 따라 시간에 비례해 에너지를 소모하고, 잔여 에너지로 남은 시간을 표현하는 시스템.**

### 추상화 모델을 이루는 요소

**상수 (Constant)**
- `CAPACITY` : 충전 가능한 에너지 최댓값 *(← 모래시계: 전체 모래량 totalSand)*
- `DRAIN_RATE` : 단위 시간당 에너지 소모량 *(← 중력에 의한 낙하 속도 flowRate)*

**변수 (Variable)**
- *독립* `energy` : 현재 보유 에너지, `0 ~ CAPACITY` (수치) — **유일한 독립 상태변수**
- *파생(읽기전용)* `remaining` = `energy ÷ DRAIN_RATE`, `progress` = `1 − energy/CAPACITY` = f(`energy`)

**상태 (State)** — 서로 배타적인 유한 집합 *(현재 어느 값인지 항상 단일 판정 가능 = 측정 가능)*
- `IDLE`(대기) · `CHARGING`(충전) · `RUNNING`(작동) · `PAUSED`(일시정지) · `DONE`(완료)

**트리거 (Trigger)** — 상태를 바꾸는 신호
- `A` 충전(외부) : `energy` 증가 (충전 중단 전까지)
- `B` 시작(외부) : 충전 종료 → 에너지 소모 시작 *(energy 값은 보존, 상태만 전환)*
- `C` 정지(외부) : 소모 중단 → `energy` 보존
- `D` 재개(외부) : 소모 재개 *(보존된 energy에서 이어서 감소)*
- `(자동)` `energy == 0` → 완료
- `RESET`(외부) : `energy = 0` → 대기

### 분해(섹션1) → 추상화(섹션3) 명칭 매핑 *(의도적 일반화)*

| 모래시계 종속 명칭 | → 도메인 중립 명칭 |
|---|---|
| `totalSand` | `CAPACITY` |
| `flowRate` | `DRAIN_RATE` |
| `topSand`(독립변수) | `energy` |
| `remaining`,`progress` | `remaining`,`progress` (의미 유지) |
| `isFlipped`(뒤집기) | 상태 전환 트리거 `B/C/D` |

**버린 것 vs 남긴 것 (의도적 제거 판단):** `flipTimestamp`, `flipEvent`, `topHeight/bottomHeight`는 `energy`·`progress`로 충분히 표현되므로 추상화 스코프에서 **제외**. 반대로 디지털 인터페이스가 새로 요구하는 `PAUSED`(일시정지) 상태와 `charged`(세션 시작 충전량)는 모래시계 원물엔 없지만 **추가**된 요소다.

### 추상화 강도 점검 — 스코프 판별 3조건
한 대상이 다음 **3조건을 모두 충족하면 스코프 내**:
**① 충전 가능한 유한 자원이 있는가 ② 시간에 비례해 단조 소모되는가 ③ 잔량으로 진행도를 표현하는가**

| 대상 | ① 충전 | ② 시간비례 소모 | ③ 잔량=진행도 | 판정 |
|---|:--:|:--:|:--:|---|
| 모래시계 / 태엽 오르골 / 기계식 스톱워치 | ✓ | ✓ | ✓ | **스코프 내** |
| 휴대폰 배터리 · 전기포트 | ✓ | ✓ | ✓ | 스코프 내 |
| 줄자 | ✗ | ✗ | ✗ | 제외 (소모·시간 개념 없음) |
| 달력 | ✗ | △ | ✗ | 제외 (충전·잔량 없음) |
| 메트로놈 | △ | ✗ | ✗ | 제외 (소모 아닌 반복 진동) |

→ 세 사물에 충실하면서도 *'충전형 시간 측정'* 목적에 딱 맞는, 과대·과소추상이 아닌 적정 강도.

## 4. 플로우차트 (Flowchart) — 새 인터페이스 **모래시계 타이머**

> 분해한 사물(모래시계)을 그대로 옮기지 않고, 위 추상화 모델에서 파생한 **새 인터페이스**를 설계한다.

### 인터페이스 개요 (한 문장)
> **모래시계의 '충전 → 소모' 원리를 빌려, 화면을 위로 드래그한 만큼 집중 시간을 충전하고, 카드를 뒤집어(Flip) 몰입 세션을 시작·정지하는 집중 타이머.**

### UI 상태 (서로 배타적)
`대기(IDLE)` → `충전 중(CHARGING)` → `집중 중(RUNNING)` ⇄ `일시정지(PAUSED)` → `완료(DONE)`

### 작동 시나리오 (조건 → 결과, *If 액션 then 상태변화*)
- **조건 1.** `[대기]`에서 카드를 **위로 드래그**하면 → `energy`가 늘고 상단 모래가 차오른다 → **`[충전 중]`**
- **조건 2.** `[충전 중]`에서 **드래그를 놓으면** → 현재 `energy`로 충전 확정 → `energy>0` 이면 **`[충전 중·준비]`**, `energy==0` 이면 **`[대기]`**
- **조건 3.** `[충전 중]`에서 카드를 **탭(뒤집기)** 하면 → 충전 종료, 모래가 흘러내려 카운트다운 시작 → **`[집중 중]`**
- **조건 4.** `[집중 중]`에서 카드를 **다시 탭(뒤집기)** 하면 → 카운트다운 정지, `energy` 보존 → **`[일시정지]`**
- **조건 5.** `[일시정지]`에서 카드를 **다시 탭** 하면 → 카운트다운 재개 → **`[집중 중]`**
- **조건 6.** `[집중 중]`에서 **`remaining == 0`** 이면 → *(내부 자동)* 모래가 전부 떨어지고 완료 알림 → **`[완료]`**
- **조건 7.** `[완료]`/`[일시정지]`에서 **리셋**을 누르면 → `energy = 0` → **`[대기]`**
- **충돌 방지 규칙.** 충전 드래그는 **위로 드래그(dy>0)** 일 때 `[대기]·[충전 중]`에서만 동작(작동/일시정지/완료에서는 충전 잠금). 아래로 드래그·기타 입력은 무시.

> ⚙️ **외부 액션 vs 내부 자동전이:** `드래그·탭·리셋`은 *사람이 주는 외부 입력*이고, **`tick`(시간 경과)** 은 *객체 내부에서 자동 발생*하는 트리거다. 조건 6의 `→[완료]` 는 사람 입력이 아닌 **(auto)** 전이.

### 상태 전이표 (모든 상태 × 모든 입력 — 충돌 없음 증명, LO3·OE4)

| 상태 ＼ 입력 | dragUp(위로 드래그) | dragEnd(놓기) | tap(뒤집기) | tick(remaining→0) *(auto)* | reset |
|---|---|---|---|---|---|
| **IDLE** | → CHARGING (energy↑) | no-op | no-op (energy=0) | N/A | → IDLE |
| **CHARGING** | energy↑ (자기전이) | energy>0 ? → CHARGING : → IDLE | → RUNNING | N/A | → IDLE |
| **RUNNING** | no-op (충전 잠금) | no-op | → PAUSED | **→ DONE (auto)** | → IDLE |
| **PAUSED** | no-op | no-op | → RUNNING | N/A | → IDLE |
| **DONE** | no-op | no-op | no-op | N/A | → IDLE |

- **no-op** = 입력은 들어오지만 의도적으로 무시(상태 불변). **N/A** = 해당 상태에선 그 입력이 발생하지 않음(예: `tick`은 RUNNING에서만 동작 — `startTick()`이 RUNNING 진입 시에만 호출).
- 모든 칸이 단 하나의 결과로 정의 → *어떤 상태에서 어떤 입력이 들어와도 충돌하는 상태 정의가 없다.*

## 5. 프로토타입 (Prototype)

플로우차트의 상태 전이 로직을 그대로 옮긴 **작동하는 웹 프로토타입**.
- 파일: `prototype.html` (의존성 없는 단일 HTML — 더블클릭으로 실행)
- **모래 물리 시뮬레이션(Canvas) — 실제 물리법칙 적용:**
  - **질량 보존** : `상부(energy) + 하부(charged−energy) = 충전량`. 충전한 만큼만 보이고, 위에서 빠진 양이 그대로 아래에 쌓인다. *조금 충전하면 시작해도 상부가 조금만 차고, 다 떨어져도 하부는 그만큼만 쌓인다.*
  - **안식각(≈32°)** : 하부 모래는 떨어진 위치(목) 아래로 안식각 원뿔 **더미**를 이루며 쌓인다(넓이가 쌓인 양에 비례하도록 계산).
  - **크레이터 vs 더미** : 빠질 땐 배출구 위에 **저압 공동(크레이터)** 이 패이고, 부을 땐 가운데가 볼록한 **더미**로 차오른다.
  - **일정 유량(Beverloo)** : 유량은 모래 높이와 무관하게 일정 → 타이머가 선형.
  - 유리·나무 프레임·모래 알갱이 질감으로 실제 사물처럼 표현.
- **콤팩트 가로 레이아웃:** 좌측에 모래시계(잔여·진행·상태를 캔버스로 시각화), 우측 슬림 레일에는 중복 없는 핵심만 — **상태 머신 strip** · 경과 시간 · 진행률 · 충전 용량. 카드 아래 **강조된 리셋 버튼**(완료 시 ‘새 세션’ 솔리드 강조, 집중 중 실수 방지 2단계 확인). 좁은 화면에서는 세로로 자동 정렬.
- 핵심 흐름: **위로 드래그(충전) → 카드 탭(뒤집어 집중 시작) → 다시 탭(일시정지/재개) → 시간 경과(완료) → 리셋**
- 스크린샷·실행 화면은 `/스크린샷` 폴더 참고.
